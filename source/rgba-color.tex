\rSec1 [rgbacolor.intro] {\tcode{rgba_color} Description}

\pnum
\indexlibrary{\idxcode{rgba_color}}%
The class \tcode{rgba_color} describes a four channel color in premultiplied format.

\pnum
There are three color channels, red, green, and blue, each of which is \tcode{double}.

\pnum
There is also an alpha channel, which is a \tcode{double}.

\pnum
Legal values for each channel are in the range $[0, 1]$.

\pnum
The type predefines a set of named colors, for which each channel is an unsigned normalized 8-bit integer.

\indexlibrary{\idxcode{rgba_color}!constructor}
\begin{itemdecl}
	rgba_color(double r, double g, double b, double a = 1.0);
  	rgba_color(double r, double g, double b, error_code& ec) noexcept;
  	rgba_color(double r, double g, double b, double a, error_code& ec) noexcept;
\end{itemdecl}
\begin{itemdescr}
	\pnum
	\requires
	\tcode{r >= 0.0} and \tcode{r <= 1.0} and \tcode{g >= 0.0} and \tcode{g <= 1.0} and \tcode{b >= 0.0} and \tcode{b <= 1.0}. Where there is an \tcode{a} parameter, \tcode{a >= 0.0} and \tcode{a <= 1.0}.
	
	\pnum
	\effects
	Constructs an object of type \tcode{rgba_color}.

	\pnum
	\postconditions
	Where there is an \tcode{a} parameter, \tcode{_Alpha = a}; otherwise \tcode{_Alpha = 1.0}.
	
	\pnum
	\tcode{_Red = r * a}.
	
	\pnum
	\tcode{_Green = g * a}.
	
	\pnum
	\tcode{_Blue = b * a}.
	
	\pnum
	\throws
	As specified in Error reporting (\ref{\iotwod.err.report}).
	
	\pnum
	\remarks
	In the event of a non-throwing error, the object shall be constructed to meet the following conditions:
	
	\pnum
	\tcode{_Red == 1.0 \&\& _Green == 0.0 \&\& _Blue == 1.0 \&\& _Alpha == 1.0}.
	
	\pnum
	\errors
	\tcode{errc::argument_out_of_domain} if the value of \tcode{r}, \tcode{g}, \tcode{b}, or \tcode{a} fails to meet the preconditions.
	
	\pnum
	\realnotes
	When an object is constructed despite the presence of one or more erroneous arguments, the resulting color is opaque magenta, which can also be obtained from \tcode{rgba_color::magenta()}.
	
	\pnum
	This color was picked because it is a legal color value, it does not have all channels set to the same value (which would hide, e.g., erroneous saturation calculations), and in the hope that it will help highlight error visibility (both visually and in an examination of the data) in the event that the developer neglects to check the \tcode{error_code\&} argument following construction of the object.
\end{itemdescr}
